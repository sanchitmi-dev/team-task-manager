Team Task Manager Demo Video Explanation
========================================

Recommended length: 2 to 5 minutes

Demo Flow
---------
1. Open the live application URL.
2. Show the login page.
3. Login as admin:
   Email: admin@example.com
   Password: Admin@12345
4. Show the dashboard:
   - Total projects
   - Total tasks
   - Completed tasks
   - Overdue tasks
   - Status overview
5. Open the Projects section.
6. Create a new project and select team members.
7. Open the Tasks section.
8. Create a task, assign it to a member, set priority, due date, and status.
9. Show the task appearing in the task board.
10. Open the Team section.
11. Show that only admin can change user roles.
12. Logout.
13. Login as member:
    Email: member@example.com
    Password: Member@12345
14. Show that member cannot create projects or manage team roles.
15. Update the status of an assigned task.
16. Return to the dashboard and show progress/status changes.

Narration Script
----------------
Hello, this is my Team Task Manager full-stack web application.

The application supports authentication with signup and login. It has role-based access control with two roles: admin and member.

I am logging in as the admin user. After login, the dashboard shows project count, total tasks, completed tasks, overdue tasks, recent tasks, and task status distribution.

As an admin, I can create and manage projects. I can add team members to a project, create tasks, assign them to members, set task priority, due date, and status.

Now I am creating a new task and assigning it to a member. The task immediately appears in the task board under its current status.

The team section is available only for admins. From here, an admin can manage user roles.

Now I will log out and log in as a member. The member view is restricted. A member cannot create projects or manage roles. The member can view assigned project tasks and update the status of their assigned tasks.

This demonstrates the main requirements: authentication, project and team management, task assignment, progress tracking, REST APIs, database persistence, validations, relationships, and role-based access control.

The app is built with Node.js, REST APIs, a JSON-backed NoSQL database, and a responsive frontend. It is ready for deployment on Railway.
